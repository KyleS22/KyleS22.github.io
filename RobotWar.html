Robot War

Robot war was created as a group project for CMPT 370 - Intermediate Software Engineering. I worked in a team of five members. The task for this project was to take a simple board game, the robot arena game, and develop a piece of software that emulated it. We studied and used the Waterfall method of development and implemented the system in java. Over the course of the semester we defined the requirements for our system, created a design document, devised a test plan, and finally implemented the system we had designed.

The game is a turn-based strategy game for 2, 3, or 6 players. Each player gets a team of three robots: one scout, one tank, and one sniper. Each type of robot has a different view range, movement count, and health. The last player to have a live robot on the board wins the game.

The major feature in our version of the system was networked mulltiplayer. I did some research on the best ways to implement networking in java and presented my findings to my team. Then a group member and I implemented the server and client system to work with our game. Adding features like fog of war and online multiplayer to a simple board game like this made a huge difference for the way people played the game. This was a very valuable experience for me.
